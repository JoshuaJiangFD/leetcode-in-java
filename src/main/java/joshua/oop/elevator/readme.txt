这一题主要首先要分析好User Cases以及具体的需求：

1） 是否只有一个电梯？
2） 楼层按钮是在外面还是电梯里面？
3） 如果有多个电梯的话，就有两种按钮，一种在电梯里面，一种在楼层上。

这里假设是一台电梯，楼层按钮在电梯里面，每一层有上下按钮，考虑调度：
有几个点：

1） 电梯内部有三个队列，当前队列，Up队列，Download队列；

2） 电梯有Up,Down两个方向，Up状态的时候根据当前楼层处理可以处理的Up请求，即放到当前队列中，同时将不能处理的Up请求放在队列中；
同理在Down状态的时候根据当前楼层处理可以处理的Down请求，即放到当前队列中，同时将不能处理的Down请求放在队列中；

3） 当遇到与当前方向相反的请求时直接放到相应方向的队列中；

4） 当处理完当前队列中的请求的时候，根据Up队列和Down队列的任务的等待情况决定电梯的方向；

5）同时为了响应外部按钮的请求，同样有两个HashMap， floor-Up 和 floor-Download，每个hashmap存储对应楼层是否有请求

电梯内部有一个thread，在响应内部当前队列的请求的同时，每到一层检查是否有该层的相同方向的请求，如果有就相应该请求
(类似于停止，开门，停顿5秒，关门)
